## 【Cursorについて】

では、今回導入したCursorについて簡単にご紹介します。

CursorはVisual Studio Codeをベースにした、AI統合型のコードエディタです。

最大の特徴は、プロジェクト全体のコードベースをインデックス化して、自然言語での検索やAIによるコーディングが可能な点です。

ProやBusinessプランでは、各種最新モデルを月500回まで高速リクエストできます。それ以降はスローリクエストまたは従量課金になります。

新しいモデルへの対応も早く、最近もclaude-4-sonnetにすぐ対応していました。

無料で使えるモデルも多数用意されているので、まずは試してみることができます。

## 【Cursorの導入】

それでは、なぜCursorを導入することになったのか、その経緯をお話しします。

導入に至った理由は2つあります。

1つ目は、Vibe codingの高まりを踏まえ、アプリケーションチームとしてAI活用を推進したかったことです。

2つ目は、個人契約で使っているエンジニアも多く、チームとして導入してほしいという要望があったことです。

導入前の環境としては、各自が好みのエディタを使用していました。VS CodeやVim、Neovimなどです。

GitHub Copilot Business経由で各種最新モデルを利用可能でしたし、個人契約でCursorを含むAIエディタを使用していたメンバーも多数いました。

## 【定着における課題：クライアントワーク】

まず、クライアントワークの特性による課題です。

少人数単位での案件進行で、案件あたり2-3名単位かつ、複数の業務が並走しています。

案件ごとに異なる開発環境で、言語やフレームワークなどが異なります。

## 【定着における課題：急速なAI環境変化】

クライアントワーク特有の課題に加えて、もう一つの大きな課題は、急速なAI環境の変化です。

エンジニア個々の嗜好や習慣があります。エディタ設定やプラグイン構成の個人最適化が進んでいますし、avante.nvimなど、各コミュニティにおけるAI IDE模倣プラグインも登場しています。

AI機能の標準化によるロックインのリスクもあります。VS Code+GitHub Copilotなど各ツールの急速な機能追従が進んでいます。Windsurfが2024年11月に、VS Code Agent Modeが2025年4月に登場しました。

codexやclaude codeなどターミナルCLIツールも次々と登場しており、選択肢が増え続けています。

## 【1ヶ月後の結果：生産性の向上】

さて、これらの課題を抱えながらも導入を進めて、1ヶ月後どうなったかをお話しします。

まず、ポジティブな面として、生産性の向上が見られました。

実装スピードが早くなりました。コード補完（tab）、自然言語によるコード生成・編集（Command K）、AIチャットによるコード相談・修正（Chatパネル）などの機能が効果的でした。

コードベース全体を対象とした質問により、仕様理解が楽になりました。

リファクタリングやテストの負担が減りました。

コーディング以外にも、ドキュメンテーション作成の補助になりました。

---

## 【1ヶ月後の結果：課題】

しかし、良いことばかりではありません。一方で、新たな課題も見えてきました。

機能認知の不足です。機能が多すぎて全てを把握しきれず、なんとなく使いこなせていない感覚に陥ることがありました。

良すぎるUXも逆に課題でした。基本機能が直感的で使いやすいため、詳細なドキュメンテーションを参照して他の機能まで深く掘り下げる必要性を感じにくいのです。

学習時間の確保も困難でした。ツールの具体的な使い方が個人の裁量に任されていたため、体系的な学習機会やサポートが必要という声もありました。

## 【実際の浸透過程】

では、こうした成果と課題を踏まえて、実際にチームにどのように浸透していったのかをお話しします。

興味深いことに、急激な変化ではなく、1ヶ月かけて徐々に浸透していきました。

Notionを活用した知見の蓄積を行いました。Tips、案件事例、他社事例リンク、課題・要望を共有しました。

複数ツール併用パターンも出現しました。Vimと同じフォルダを並列で開いて、コードベース全体への質問など特定の場面でCursorを活用する人もいました。

特定の言語やプロジェクトでのみCursorを使用するパターンもありました。PythonやTypeScriptはLLM自体の学習が進んでいるので使いやすいのですが、GoやPHPはあまり効果を感じないという声もありました。そのため、rulesをちゃんと書いた方がいいという知見も得られました。

## 【VS Codeへの収束傾向】

ここで、少し視点を変えて、業界全体の最近の動向についてもお話ししたいと思います。

最近の動向として、VS Codeへの収束傾向が見られます。

急速な機能追従が進んでいます。Cursorが2024年7月にComposer（マルチファイル同時編集機能）をリリースすると、GitHub Copilotも2024年10月にMulti-file editingをリリースしました。

Cursor Agent Modeが2024年9月にリリースされると、VS Code Agent Modeが2025年4月にリリースされました。

一方で、CursorはVS Code 1.96（2024年11月）を基盤としており、基盤バージョンの遅延による互換性問題も発生しています。

Microsoft製拡張機能のフォークIDE非対応化が2025年4月に発表され、VS CodeのGitHub Copilot Chat拡張のOSS化が2025年5月19日に発表されました。
